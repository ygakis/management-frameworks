Layer 1 — Strategic Direction
Acronym What it is Origin Who uses it AI-Native Equivalent
OKRObjectives &
Key Results		 Set a qualitative Objective (direction) + 3–5 measurable Key Results that prove you got there. KRs are outcomes, never tasks. Reviewed quarterly. Andy Grove / Intel 1970s. John Doerr brought to Google 1999. Tech CEOs, startups, product teams. Now spreading to all industries. System prompt + success criteria. The Objective = agent goal; KRs = verification conditions Claude checks against.
KPIKey Performance
Indicator		 Ongoing vital signs of the business. Revenue, churn, conversion, margin. Always-on metrics, not change targets. KPIs tell you if the machine is healthy; OKRs tell you where you're going. Management science, 1970s. Peter Drucker lineage. Every business, every level. Finance, ops, marketing, sales. Evaluation metrics in AI benchmarking. The numbers you define to assess whether an AI workflow is performing.
OODAObserve · Orient
Decide · Act		 Decision loop. Whoever cycles through it faster gains advantage. Orientation is the critical step — it filters what you observe through mental models and experience. Disrupting your opponent's loop is strategy. Col. John Boyd, USAF, 1960s–70s. F-86 vs MiG-15 analysis. Military, competitive strategy, trading, crisis management. Agentic reasoning loop. Real-time AI agents operate as OODA machines — observe context, orient via model, decide action, act via tool use.
JTBDJobs To
Be Done		 People don't buy products — they hire them to do a job. "What job is this customer firing their old solution to hire mine for?" Reframes product thinking from features to purpose. Clayton Christensen, Harvard. Popularized in The Innovator's Dilemma, 1997. Product managers, marketers, brand strategists, founders. Prompt intention modeling. Before writing a prompt, ask: what job is this prompt hired to do? Prevents feature-thinking ("add more detail") vs. outcome-thinking.
Layer 2 — Prioritization
Acronym What it is Origin Who uses it AI-Native Equivalent
RICEReach × Impact ×
Confidence ÷ Effort		 Scoring model for ranking competing initiatives. Reach = users affected/period. Impact = 0.25–3 scale. Confidence = % certainty in estimates. Effort = person-months. Forces explicit, comparable assumptions. Sean McBride, Intercom, ~2016. Product managers, growth teams, startup founders. Prompt ROI assessment. Before building an AI workflow, score it: how many tasks does it reach, what's the impact, how confident are you it'll work, how long to build.
ICEImpact ·
Confidence · Ease		 Lighter RICE. Three factors, no Reach. Fast relative ranking. Good for solo operators and small teams where precision is less important than speed of decision. Sean Ellis (growth hacking community), ~2015. Solo founders, growth hackers, small teams, content strategists. Quick prompt triage. Rate each AI task idea: will it matter (I), does it work reliably (C), how fast to implement (E). Your daily filter.
MoSCoWMust · Should ·
Could · Won't		 Requirements categorization. The discipline is committing to "Won't Have" — explicit de-scoping. Prevents scope creep by making trade-offs visible. Run at project start, revisit each sprint. Dai Clegg, Oracle UK, 1994. Part of DSDM framework. Project managers, business analysts, product teams, clients. Constraint declaration in prompts. Explicitly stating Won't = better outputs. "Do not include X, Y, Z" is MoSCoW applied to AI instruction design.
WSJFWeighted Shortest
Job First		 Enterprise-grade prioritization. Score = Cost of Delay ÷ Job Size. Prioritizes high-value, fast-to-deliver items. Accounts for the cost of waiting, not just value of doing. Don Reinertsen, Lean product development. Used in SAFe framework. Large Agile organizations, enterprise PMs, program-level planning. Workflow sequencing for AI pipelines. Which automation to build first based on time-cost of doing it manually vs. build effort.
Layer 3 — Execution Methodology
Acronym / Method What it is Origin Who uses it AI-Native Equivalent
AgilePhilosophy, not
an acronym		 Manifesto-level philosophy: ship working increments fast, adapt via feedback, prefer collaboration over documentation. A reaction against Waterfall rigidity. Scrum and Kanban are implementations of it. Agile Manifesto, Snowbird Utah, Feb 2001. 17 software practitioners. Software teams universally. Now spreading into marketing, operations, design. Iterative prompting. Don't prompt for the final output — prompt for drafts, review, iterate. Agile applied to human-AI co-creation.
ScrumRoles · Sprints
· Ceremonies		 Agile's structured implementation. Roles: Product Owner (what to build), Scrum Master (remove blockers), Dev Team (build). Rhythm: Sprint (2-week cycle), Daily Standup (15 min sync), Sprint Review (demo), Retrospective (learn). Ken Schwaber & Jeff Sutherland, 1990s. Named after rugby formation. Software dev teams, product companies, increasingly marketing and ops. Structured AI session design. Sprint = context window. Standup = status prompt. Review = verification step. You as Product Owner, Claude as Dev Team.
KanbanVisual pull
system		 Visual board: To Do → In Progress → Done. No fixed sprints — continuous flow. Manage by limiting work-in-progress (WIP limits). Pull new work only when capacity opens. Less ceremony than Scrum. Toyota Production System, 1940s. Taiichi Ohno. Adapted to software by David Anderson, 2004. Operations teams, service teams, solo operators, support workflows. Task queue management for AI agents. Limit concurrent agent tasks. Visual tracking of which automations are in progress vs. queued.
Design ThinkingEmpathize · Define
Ideate · Prototype · Test		 Human-centered problem solving. Start with Empathy (understand the real user), Define the actual problem, Ideate broadly, Prototype fast, Test with real users. Bias toward action and iteration over analysis paralysis. IDEO, Stanford d.school. David Kelley, 1980s–90s. Designers, product teams, innovation consultants, strategy consultants. Prompt design methodology. Empathize = who is the end user of this output? Define = what's the real job? Prototype = test prompt variants. Test = eval with real examples.
WaterfallSequential
phases		 Classic linear model: Requirements → Design → Build → Test → Deploy. Each phase complete before the next begins. Works when requirements are stable (construction, manufacturing). Fails when they change (software, creative work). Winston Royce, 1970. Originally described as flawed — the industry adopted it anyway. Construction, government contracts, regulated industries, hardware. Single-shot prompting. Define everything upfront, generate once, deliver. Valid for stable, well-specified tasks. Fails for complex, exploratory AI work.
Layer 4 — Process Documentation & Coordination
Acronym What it is Origin Who uses it AI-Native Equivalent
SOPStandard Operating
Procedure		 A repeatable recipe for a known, solved problem. Write it once, replicate the outcome without rethinking. Encodes institutional knowledge. Protects against key-person dependency. The foundation of operational scale. Military/industrial management. W. Edwards Deming lineage. Pre-WWII origins. Operations managers, quality control, manufacturing, healthcare, any repeatable process. Reusable system prompt / skill file. Your Claude Code skills and SOP.md files are exactly this — write the procedure once for the AI to follow consistently.
PRDProduct Requirements
Document		 Upstream document that defines what to build and why before building starts. Covers: problem statement, user needs, success criteria, scope, constraints, non-goals. Prevents building the wrong thing with precision. Silicon Valley product management culture, 1980s–90s. Formalized at companies like Apple and Netscape. Product managers, founders, engineering leads, startup CTOs. System prompt for a product. A PRD written clearly is a near-complete brief for an AI to build or spec a product. Also: the design doc for AI workflow architecture.
RACIResponsible ·
Accountable ·
Consulted · Informed		 Coordination matrix. For every task/decision: R = who does the work, A = who owns the outcome (one person), C = who gives input, I = who gets notified. Eliminates ambiguity about who decides. "Too many A's" = organizational dysfunction. Management consulting, 1950s–60s. Popularized in project management literature. Project managers, operations leads, consultants, HR, any cross-functional work. Human-AI task assignment protocol. For each workflow: R = Claude agent, A = you as CEO, C = tools/MCPs consulted, I = outputs delivered. Design before you build.
Layer 5 — AI-Native Cognitive Loops
Acronym What it is Origin Who uses it AI-Native Equivalent
PDCAPlan · Do
Check · Act		 The original iterative improvement loop. Plan what you'll do. Do it. Check results against expectations. Act on what you learned — standardize if it worked, adjust if it didn't. Every Agile sprint is a PDCA cycle. Foundation of quality management. W. Edwards Deming (called it Shewhart Cycle). Rebuilt Japanese manufacturing post-WWII. Quality managers, operations teams, manufacturing, process improvement everywhere. This IS the AI-native loop. Prompt (Plan) → Generate (Do) → Evaluate output (Check) → Refine prompt (Act). Your iterative prompting discipline, formalized.
E·P·E·VExplore · Plan
Execute · Verify		 Agentic AI execution protocol. Explore = read codebase/context before acting. Plan = think through approach, declare steps. Execute = make changes. Verify = confirm outputs match intent. Prevents the AI equivalent of cutting before measuring. Emergent from LLM/agent engineering practice, 2023–24. Claude Code, Cursor, agentic frameworks. AI engineers, developers using Claude Code / Cursor / Copilot Workspace, prompt architects. Native. This is PDCA at AI timescale — one context window instead of one quarter. Design your Claude Code tasks to explicitly trigger each phase before the next.
OODA(repeated at
AI layer)		 At the AI layer, OODA describes real-time agent reasoning. The agent observes the environment (tool outputs, file states), orients via its model, decides the next action, acts via tool call. Faster loop = more adaptive agent. Disrupting the loop = hallucination or context loss. Col. John Boyd (see Layer 1) — but its AI application is fully emergent, 2023–present. AI researchers, agent architects, anyone building multi-step agentic workflows. Native at this layer. Design agentic workflows to maintain loop integrity: don't interrupt Orientation with conflicting context. Context window hygiene = loop hygiene.
